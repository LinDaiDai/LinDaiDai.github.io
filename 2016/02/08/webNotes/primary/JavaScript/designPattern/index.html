第一章.设计模式简介

作用：JavaScript设计模式的作用 - 提高代码的重用性，可读性，使代码更容易的维护和扩展。

一般只用于大型的项目.

设计模式有六大原则：

  1. 开闭原则。就是说模块应对扩展开放，而对修改关闭。
  2. 里氏代换原则。如果调用的是父类的话，那么换成子类也完全可以运行。
  3. 依赖倒转原则。把父类都替换成它的子类，程序的行为没有变化。
  4. 接口隔离原则，每一个接口应该是一种角色，不多不少，不干不该干的事，该干的事都要干。
  5. 单一职责原则。
  6. 迪米特法则。 最少知识原则。
